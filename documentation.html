Documentation

Full system design and technical reference for WitcherCraft.

1. Overview

Witchercraft is a Minecraft mod built on NeoForge that brings the world of CD Projekt Red's The Witcher 3: Wild Hunt into the Minecraft engine. The mod aims to recreate the core systems that define the Witcher experience — Signs, Alchemy, monster lore, and a deep RPG progression system — while adapting them thoughtfully to Minecraft's design language.

Rather than simply porting assets, Witchercraft treats each system as a design problem: how do you make Witcher mechanics feel meaningful and balanced inside a sandbox game that operates by fundamentally different rules? The result is a mod that respects both source materials — staying faithful to Witcher 3's feel while remaining coherent within Minecraft's world.

More powerful than potions with higher toxicity cost. Each is derived from a monster and grants abilities inspired by that creature: Ekimmara (life steal), Katakan (crit chance), Troll (out-of-combat regen), Werewolf (in-combat regen), Wyvern (attack speed), Succubus (escalating damage), Water Hag, Grave Hag, Foglet, Leshen, Nekker Warrior, Wraith.

Oils

Applied to weapons as enchantments, targeting specific enemy archetypes. Beast, Necrophage, Specter, Vampire, Insectoid, Ogroid, Draconid, Relict, Cursed, Hanged Man's Venom.

4. Technical Design

This section covers the key engineering and design decisions behind Witchercraft — the problems each system solves and the rationale behind the chosen approaches.

4.1 Damage Model & Formula

Witchercraft replaces Minecraft's default damage calculation with a custom DamageCalculatorProcedure that fires on every player attack. The system introduces flat additional damage, percentage increased damage, probabilistic critical hits, critical multiplier, and life steal.

Variables
VariableDescription
baseDamageRaw damage value of the weapon or attack source
additionalDamageFlat bonus damage added before any multipliers (from skill tree or gear)
increasedDamagePercentage damage bonus, stored as an integer (e.g. 10 = +10%)
critChanceInteger 1–100. Rolled against a random number each hit
critDamageCrit damage multiplier stored as integer (e.g. 150 = ×1.50)
lifeStealPercentage of final damage restored as health (e.g. 10 = 10%)
Normal Hit
// Normal hit
finalDamage = (baseDamage + additionalDamage) × (1 + increasedDamage × 0.01)
healthRestored = finalDamage × lifeSteal × 0.01
Critical Hit

A critical hit occurs when a random integer rolled between 1 and 100 is less than or equal to the player's critChance. On a crit, the damage is further multiplied by the critDamage multiplier:

// Crit check
critRoll = random(1, 100)

if critRoll <= critChance:
  finalDamage = (baseDamage + additionalDamage)
                × (1 + increasedDamage × 0.01)
                × (critDamage × 0.01)
  healthRestored = finalDamage × lifeSteal × 0.01

All percentage stats are stored as integers and multiplied by 0.01 at calculation time — avoiding floating point drift and keeping HUD values human-readable. The Ender Dragon is explicitly excluded from the critical hit path to prevent one-shot boss scenarios. Damage and life steal resolve via a one-tick server-side delay, preventing race conditions on the player's health value in multiplayer.

4.2 Toxicity System

⬜
[ IMAGE: HUD showing Toxicity rising after consecutive potion use ]

Figure 4.1 — Toxicity level rising on the HUD after consecutive potion consumption.

  • Every potion and decoction consumed increments current Toxicity by a fixed item-specific amount.
  • Decoctins carry a higher toxicity cost than standard potions, reflecting their greater power.
  • Toxicity decays passively over time between potion uses.
  • If current Toxicity exceeds Maximum Toxicity, health drains each tick until it falls back below the threshold.
  • White Honey immediately clears all active effects and resets Toxicity to zero.
  • Both values are displayed on the HUD at all times.

The system exists to prevent the dominant strategy of stacking all potions before every encounter. Without it, optimal play is always maximum potion stacking, which trivialises the Alchemy system. Maximum Toxicity can be raised through Acquired Tolerance upgrades in the Alchemy Skill Tree, rewarding investment in the Alchemy path with greater flexibility.

4.3 Skill Tree Architecture & Perks

Four independent tabs, each corresponding to a different combat philosophy. All skill state is stored as persistent player data and synchronised client-server via custom network packets. Perks are arranged in tiers — basic perks are accessible immediately, while higher-tier perks require prior investment in the same tab.

4.4 Developer & Debug Commands

Custom commands registered for development and debugging. The Dev Log command is particularly valuable — it exposes the full damage calculation pipeline in real time via chat messages, making it possible to verify that stat changes from skill upgrades, gear, and potions apply correctly.

/witchercraft statsPrints all current player variables — Toxicity, crit chance, crit damage, life steal, etc. — to chat
/witchercraft levelfixCorrects any desync in player level or XP values that can occur during testing
/witchercraft devlogToggles Dev Log status effect. With it active, every damage calculation prints its full breakdown to chat — base damage, combined damage, crit roll, crit result, and life steal
/witchercraft resetskillsResets all Skill Tree investments, returning spent points for redistribution
/witchercraft settoxicity [value]Directly sets current Toxicity to the specified value for testing threshold behaviours

6. Installation

  • Download and install NeoForge from the official NeoForge installer.
  • Place the WitcherCraft .jar file into your Minecraft /mods/ folder.
  • Launch Minecraft using the NeoForge profile.
  • Press Tab to open the Sign Menu and B to open Alchemy. Access Character, Skill Tree, Bestiary and Meditation from the Pause Menu.
